# Security Policy

## Supported Versions

boto3-s3 is in beta, preparing for the 1.0 release. Only the latest release of
each package — `boto3-s3` and `boto3-s3-cli` — receives security fixes; fixes
are not backported to earlier versions.

## Reporting a Vulnerability

Please report security vulnerabilities **privately** — do not open a public
issue, pull request, or discussion for them.

Use GitHub's private vulnerability reporting: open the repository's
**[Security](https://github.com/izumo-m/boto3-s3/security/advisories)** tab and
click **"Report a vulnerability"**. The report stays visible only to the
maintainers.

A helpful report includes:

- the affected package and version (`boto3-s3` or `boto3-s3-cli`);
- the Python, `boto3` / `botocore`, and operating-system versions;
- steps to reproduce — ideally a minimal proof of concept;
- the impact you expect.

This is a personal, volunteer-run project, so handling is best-effort. Confirmed
issues are coordinated and disclosed through a GitHub Security Advisory, and
reporters are credited there unless they ask otherwise.
